## 🎯 项目简介

NO-CRM 是一个轻量级客户关系管理系统，专为小团队和个体销售设计。系统采用前后端分离架构，使用 JSON 文件作为数据存储，无需复杂的数据库配置，开箱即用。

### 核心功能

- ✅ 用户认证与权限管理
- ✅ 客户信息管理
- ✅ 销售线索跟踪
- ✅ 跟进记录管理
- ✅ 任务提醒功能
- ✅ 数据可视化大屏

## 🚀 快速开始

### 环境准备

1. **安装 Node.js**（建议 v16 或更高版本）
2. **安装 npm**（通常随 Node.js 一起安装）

### 启动步骤

#### 1. 克隆项目（或解压）

```bash
cd NO-CRM
```

#### 2. 安装后端依赖

```bash
cd backend
npm install
```

#### 3. 启动后端服务

```bash
npm run start:dev
```

后端服务将在 http://localhost:3000 启动

#### 4. 安装前端依赖（新开一个终端）

```bash
cd frontend
npm install
```

#### 5. 启动前端服务

```bash
npm run dev
```

前端服务将在 http://localhost:5173 启动

#### 6. 访问系统

打开浏览器访问 http://localhost:5173

### 首次使用

1. 点击"注册"标签页
2. 填写用户信息（建议先注册一个管理员账号）
3. 登录后即可使用所有功能

## 🔑 核心概念

### 1. 数据模型

系统中有 5 个核心数据模型：

- **User（用户）**：系统用户，包含账号、密码、角色等
- **Customer（客户）**：客户信息，包含姓名、电话、公司等
- **Lead（线索）**：销售线索，包含状态、意向等级等
- **Activity（跟进记录）**：与客户的沟通记录
- **Task（任务）**：待办事项和提醒

### 2. 权限控制

- **管理员（admin）**：可查看和管理所有数据
- **销售（sales）**：只能查看和管理自己的数据

### 3. 数据存储

采用 JSON 文件存储，简单轻量：
- 数据文件位于 `data/` 目录
- 每个模块一个 JSON 文件
- 支持 CRUD 操作
- 自动创建和更新

## 📝 开发规范

### 代码风格

- 使用 TypeScript 编写代码
- 遵循 ESLint 规则
- 使用有意义的变量名
- 添加必要的注释

### Git 提交规范

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构代码
test: 测试相关
chore: 构建/工具相关
```

### 目录命名

- 使用小写字母和中划线：`customer-list`
- 模块文件夹使用复数：`customers/`
- 组件使用 PascalCase：`CustomerList.vue`
